Introducción¶

Recordemos los dos tipos de aprendizaje automtico:

  • aprendizaje supervisado

  • aprendizaje no supervisado

La diferencia entre ambos radica en la existencia o no de una variable objetivo para predecir.

  • Los mtodos supervisados requieren tanto de datos de input como el output (o target) asociado para generar las reglas.

  • Los mtodos no supervisados solo requieren datos de input.

Aprendizaje supervisado¶

Los algoritmos de aprendizaje supervisado se dividen en dos grandes tipos:

  • Regresiones, las que ya estuvimos estudiando.

  • Clasificaciones, las que veremos en detalle en estas clases.

Clasificación¶

Con la clasificacin, buscamos predecir una variable objetivo que es categrica en lugar de continua.

Algunos ejemplos tpicos:

  • Prediccin de bajas de clientes a la suscripcin de un servicio (modelos de churn)
  • Distincin de comentarios positivos y negativos en redes sociales
  • Filtros de spam en servicios de correo electrnico
  • Diagnstico de enfermedades
  • Deteccin de fraudes de tarjeta de crdito

Existen diferentes modelos que implementan soluciones a problemas de clasificacin.

KNN - K Nearest Neighbours

KNN¶

Comenzamos con una tcnica simple de clasificacin conocida como k - Nearest Neighbors (KNN) (vecinos ms cercanos).

Informalmente, se puede explicar con la frase dime con quin andas y te dir quin eres.

Bsicamente, los pasos de KNN son:

  1. Memoriza la ubicacin de cada observacin del train dataset, segn los valores de sus features.

  2. Cuando recibe un dato nuevo, lo ubica tambin en el espacio.

  3. Encuentra los k vecinos ms cercanos (k es un hiperparmetro del modelo).

  4. Cada uno de ellos aporta un "voto" a la clase a la que pertenece.

  5. La prediccin para el dato nuevo queda determinada por la clase mayoritaria entre los "votos" del paso anterior.

KNN¶

K- Nearest Neighbors:

  • Crea un espacio N dimensional, donde cada dimensin es una de las n features. Cada observacin se ubica y memoriza en ese espacio.

  • Es un algoritmo lazy (perezoso), ya que no aprende una funcin discriminante en la etapa de entrenamiento (no genera un modelo).

  • Todas las observaciones se usan al momento de prediccin, por lo que es costosa en trminos computacionales. Y aumenta ese costo al incrementar la cantidad K de vecinos ms cercanos.

  • Generalmente funciona con features continuas, aunque se puede modificar para trabajar con features categricas.

  • Pertenece a la clase de modelos llamados IBL: Aprendizaje Basado en Instancias (Instance Based Learning).

Evaluación del modelo¶

Otra forma ms precisa de evaluar los modelos es usando cross-validation.

Generamos nuevamente modelos con una secuencia de valores para el hiperparmetro n_neighbors entre 1 y 25.

Pero ahora Cross Validation devuelve tantos resultados o scores como folds le indicamos. En este caso, por cada valor del hiperparmetro n_neighbors, cv devuelve cinco scores, ya que seleccionamos 5 folds.

Luego calculamos la media de los scores. Por defecto KNN usa accuracy.

In [13]:
from sklearn.model_selection import cross_val_score, KFold

# Usamos cross validation con 5 folds
kf = KFold(n_splits=5, shuffle=True, random_state=12)

scores_para_df = []

for i in range(1, 26):
    
    # En cada iteracin, instanciamos el modelo con un hiperparmetro distinto
    model = KNeighborsClassifier(n_neighbors=i)
    
    # cross_val_scores nos devuelve un array de 5 resultados,
    # uno por cada particin que hizo automticamente CV
    cv_scores = cross_val_score(model, X_train, y_train, cv=kf)
    
    # Para cada valor de n_neighbours, creamos un diccionario con el valor
    # de n_neighbours y la media de los scores
    dict_row_score = {'score_medio':np.mean(cv_scores), 'n_neighbors':i}
    
    # Guardamos cada uno en la lista de diccionarios
    scores_para_df.append(dict_row_score)

Ajuste del modelo

Ajuste del modelo¶

La capacidad de generalizacin de un modelo depende de un buen ajuste a los datos de entrenamiento.

  • Si seleccionamos un K muy bajo, por ejemplo K=1, el modelo ser muy complejo. Tenemos una situacin de overfit. Con alta varianza y bajo sesgo.

Tambin el modelo tiende a ser sensible a outliers.

  • Si seleccionamos un K muy alto, el modelo ser demasiado simple. Caemos en una situacin de underfit. Con baja varianza y alto sesgo.

Ajuste del modelo¶

Con KNN, una forma de equilibrar el trade-off entre sesgo y varianza es probar distintos valores del hiperparmetro k hasta encontrar el adecuado.

Consideremos la clase de cada observacin con dos valores: verde y naranja.

Cada rea indica el color a predecir para las observaciones que caen en ella.

  • K=1 Cada rea se ajusta a los datos de su clase. Es un sobreajuste.

  • K=3 Es el modelo correcto. Resuelve el dilema entre sesgo y varianza.

  • K=7 Se observan practicamente solo dos reas; genera un modelo muy simple.

Distancias¶

Finalmente, estudiemos en detalle las distancias, el concepto que permite a KNN determinar la cercana entre las observaciones.

Por qu distancias? No hay una sola?

Distancias¶

Existen muchas medidas o mtricas de distancias. Por ejemplo, algunas de ellas:

Similaridad¶

Las mtricas de similaridad son una forma de medir lo "parecidos" que son los datos entre s.

Usualmente se expresan con valores numricos, entre 0 y 1. 0 indica baja similaridad, 1 muy similar.

Se aplican sobre datos numricos, alfabticos, etc.

Un ejemplo de similaridad es el coeficiente de Jaccard:

Ejemplo en Python:

In [26]:
def jaccard_similarity(list1, list2):
    s1 = set(list1); s2 = set(list2)
    return float(len(s1.intersection(s2)) / len(s1.union(s2)))

list1 = ['dog', 'cat', 'cat', 'rat']
list2 = ['dog', 'cat', 'mouse']
jaccard_similarity(list1, list2)
Out[26]:
0.5

Distancias (definición)¶

Una mtrica d es una distancia si cumple con:

  1. No negativa. d(p, q) 0, para todas observaciones distintas p y q.
  2. Simetra. d(p, q) = d(q, p) para todo p,q.
  3. Desigualdad triangular: d(p, q) d(p, r) + d(r, q) para todo p,q,r.
  4. Igualdad. d(p, q) = 0 si y solo si p = q.

Distancia Euclidea¶

Es la medida de distancia ms comn e intuitiva.

Se puede pensar a uno de los puntos como el centro de una crculo, y la medida es el radio.

Trabaja bien con datos con baja dimensionalidad. Es la distancia por defecto que usa KNN.

Se define: dados dos puntos $X = (x_1, x_2, ..., x_n)$ e $Y = (y_1, y_2, ..., y_n)$ :

$$ D_{(X, Y)} = \sqrt{\sum_{i=1}^{n} (x_i - y_i)^2}$$

Distancia de Manhattan¶

Tambin llamada geometra de taxista o distancia de manzana de ciudad por su apariencia similar al recorrido de un taxista en las calles de Manhattan.

Aqu tenemos en lugar de un crculo, un cuadrado. No se permiten movimientos diagonales para calcular la distancia.

Parece trabajar bien con datos con alta dimensionalidad. La desventaja es que siempre es mayor a la distancia euclidea.

Dados dos puntos $X = (x_1, x_2, ..., x_n)$ e $Y = (y_1, y_2, ..., y_n)$ :

$$ D_{(X, Y)} = \sum_{i=1}^{n} |x_i - y_i| $$

La distancia entre los dos puntos es la suma de las diferencias (absolutas) de sus coordenadas.

Distancia de Chebyshev¶

Es la mxima distancia en cualquiera de sus ejes.

Es decir, si cada punto lo vemos como un vector con N coordenadas, se considera la distancia de Chebyshev la mayor de todas las distancias en cada eje.

Podemos comparar las tres distancias:

Solo se usa para casos especficos, donde estamos seguros que dar buenos resultados.

Dados dos puntos $X = (x_1, x_2, ..., x_n)$ e $Y = (y_1, y_2, ..., y_n)$ :

$$ D_{(X, Y)} = \max_{i} |x_i - y_i| $$

Distancia de Minkowski¶

Es una mtrica en un espacio vectorial normalizado.

Se puede considerar una generalizacin de las otras distancias.

Se define para dos puntos $X = (x_1, x_2, ..., x_n)$ e $Y = (y_1, y_2, ..., y_n)$ como:

$$ D_{(X, Y)} = (\sum_{i=1}^{n} |x_i - y_i|^p)^{\frac{1}{p}} $$

Lo interesante es el uso del parmetro p:

  • p=1. Es la distancia de Manhattan.
  • p=2. Es la distancia Euclideana.
  • p=. Es la distancia de Chebyshev.

Conclusiones

Conclusiones¶

El modelo K- Nearest Neighbors - KNN - es uno de los algoritmos ms sencillos para clasificar.

  • Permite clasificar a partir de la cercana con otras observaciones del dataset de entrenamiento.

  • No genera un modelo, la prediccin se realiza usando todas las observaciones.

  • K es el hiperparmetro del modelo, que es nmero de vecinos a considerar.

  • Para validar el modelo, debemos variar K y medir su performance. Es conveniente usar cross validation.

  • Las medidas de distancia es lo que determina la cercana entre los datos. Existen distintas tipos de distancias.

    for i in range(start, stop):
    
        # En cada iteracin, instanciamos el modelo con un hiperparmetro distinto
        model = KNeighborsClassifier(metric=metrica,n_neighbors=i)
    
        # cross_val_scores nos devuelve un array de 5 resultados,
        # uno por cada particin que hizo automticamente CV
        cv_scores = cross_val_score(model, X_train, y_train, cv=kf)
    
        # Para cada valor de n_neighbours, creamos un diccionario con el valor
        # de n_neighbours y la media de los scores
        dict_row_score = {'score_medio':np.mean(cv_scores), 'n_neighbors':i}
    
        # Guardamos cada uno en la lista de diccionarios
        scores_para_df.append(dict_row_score)

    # Creamos el DataFrame a partir de la lista de diccionarios
    df_scores = pd.DataFrame(scores_para_df)
    
    # Retornamos el DF
    return df_scores
