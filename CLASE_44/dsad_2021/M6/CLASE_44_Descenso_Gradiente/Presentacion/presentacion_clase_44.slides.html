Introduccin

¿Qué es gradiente?¶

El gradiente es la derivada o tasa de cambio de una funcin. Es un vector (una direccin para moverse) que:
  • Apunta en la direccin de mayor aumento de una funcin
  • Es cero en un mximo o un mnimo local (porque no hay una nica direccin de aumento)
El trmino "gradiente" se utiliza normalmente para funciones con varias variables y una nica salida (un campo escalar). Podemos hablar del gradiente de una recta (su pendiente), pero usar "gradiente" para funciones de una sola variable es innecesariamente confuso.

¿Qué es descenso gradiente?¶

Es un algoritmo de optimizacin genrico para encontrar el mnimo de una funcin.
Es un algoritmo de optimizacin iterativo, comenzamos con un punto aleatorio en la funcin y nos movemos en la direccin negativa del gradiente de la funcin para alcanzar los mnimos locales / globales.
En machine learning buscamos mnimos de la funcin de costo.
La funcin de costo (J) o funcin de prdida mide la diferencia entre el valor real y la prediccin del modelo. La funcin de costo es una funcin convexa.
Descenso gradiente

¿Cuándo usamos descenso gradiente?¶

Generalmente utilizamos descenso de gradiente cuando los parmetros del modelo no se pueden calcular analticamente (por ejemplo, utilizando lgebra lineal) y deben buscarse mediante un algoritmo de optimizacin.

Cuando nuestro modelo tiene muchas variables, muchas observaciones o cuando su funcin de costo no puede resolverse de forma analtica (como es el caso de la regresin logstica y las redes neuronales), podemos entrenar nuestro modelo usando el algoritmo descenso gradiente.

Funcin de Costo

La funcin de costo de la regresin lineal es una funcin convexa, lo que nos asegura que el descenso del gradiente va a converger al mnimo global.

En las funciones de costo que con ms de un mnimo, como la de la funcin logstica, cuando ejecutamos el/los algoritmos de descenso del gradiente buscamos adems iniciarlos desde diferentes lugares/posiciones para que no quede atrapado en un mnimo local.

Ejemplos de funciones de costo que conocemos¶

Regresión lineal¶

La funcin de costo (J) de la regresin lineal es el error cuadrtico medio (RMSE) entre el valor y predicho (pred) y el valor y verdadero (y).

Supongamos que el dataset tiene n filas y m columnas.

Para la fila i del dataset, el valor predicho $h_i$:

$$h_i = \beta_0 + \beta_1. X_{i1} + \beta_2. X_{i2} + \beta_3. X_{i3} + ... + \beta_m. X_{im}$$

Costo:

$$J = \frac{1}{n} \sum_{i=1}^n{(h_i - y_i)^2}$$

Gradiente:

$$\frac{\partial J(\beta_0, ..., \beta_m)}{\partial \beta_j} = \frac{2}{n} \sum_{i=1}^n (h_i - y_i). X_{ij} $$

Regresión logística¶

$$\beta_0 + \beta_1 . x_1 + ... + \beta_m. x_m = \beta^t.X$$

$$h_{\beta}(X) = g(\beta^t.X)$$

$$g(z) = \frac{1}{1+e^{-z}}$$

Costo:

$$J = - \space \frac{1}{n} \sum_{i=1}^n(y_i . log(h_{\beta}(X_i)) + (1-y_i) . log(1-h_{\beta}(X_i)))$$

Gradiente:

$$\frac{\partial J(\beta_0, ..., \beta_m)} {\partial \beta_j} = \sum_{i=1}^n(h_{\beta}(X_i) - y_i).X_{ij}$$

Algoritmo

Algoritmo¶

El algoritmo comienza estableciendo valores iniciales para el/los coeficiente/s de la funcin.

Estos valores pueden ser 0 o valores aleatorios pequeos.

coefficient = 0.0

El costo para estos valores en los coeficientes se calcula evaluando la funcin de costo en los valores iniciales.

cost = f(coefficient)

cost = evaluate(f(coefficient))

Se calcula la derivada de la funcin de costo.

La derivada es un concepto de clculo que refiere a la pendiente de la funcin en un punto dado.

Necesitamos conocer la pendiente para determinar la direccin (signo) en que debemos variar los valores de los coeficientes para que el costo en la siguiente iteracin sea menor.

delta = derivative(cost)

Ahora que sabemos por la derivada qu direccin es "cuesta abajo", podemos actualizar los valores de los coeficientes.

Debemos especificar el valor de un parmetro de tasa de aprendizaje (learning rate) $\alpha$ que controla cunto pueden cambiar los coeficientes en cada iteracin.

coefficient = coefficient  (alpha * delta)

Este proceso se repite hasta que el costo sea 0 o suficientemente cercano a 0 para ser aceptable.

Learning rate¶

La tasa o velocidad de aprendizaje (learning rate) $\alpha$ es un hiperparmetro muy importante del descenso gradiente.

Para reducir el tiempo de convergencia del descenso del gradiente, es conveniente que las variables estn normalizadas.

Una buena forma de asegurarse de que el descenso gradiente se ejecute correctamente es graficando los valores de la funcin de costo mientras se ejecuta la optimizacin.

Graficando el nmero de iteraciones en el eje x y el valor de la funcin de costo en el eje y, podemos ver el valor de la funcin de costo despus de cada iteracin de descenso de gradiente, y detectar fcilmente qu tan apropiada es la tasa de aprendizaje.

Stopping criteria

Stopping criteria¶

Algunos criterios de terminacin del algoritmo de descenso gradiente son:

  • Cantidad mxima de iteraciones

  • Tolerancia absoluta: se detiene cuando el valor de la funcin de costo se "acerca lo suficiente" a cero.

  • Tolerancia relativa: se detiene cuando la mejora en esa iteracin cae por debajo de un umbral.

Conclusiones¶

Hands-on

Hands-on¶

Vimos que el gradiente para una regresin lineal se calcula como

$$\frac{\partial J(\beta_0, ..., \beta_m)}{\partial \beta_j} = \frac{2}{n} \sum_{i=1}^n (h_i - y_i). X_{ij} $$

El algortimo actualiza los coeficientes como

$$\beta_j = \beta_j - \alpha . \frac{\partial J(\beta_0, ..., \beta_m)}{\partial \beta_j} $$

Queda:

$\beta_0 = \beta_0 - \alpha \frac{2}{N} \sum (h_i - y_i)$

$\beta_i = \beta_i - \alpha \frac{2}{N} \sum (h_i - y_i). X_i$

Presentamos a continuacin una clase que resuelve el algoritmo de descenso gradiente para una regresin lineal simple.

class MyGradientDescent():
    def __init__(self, learning_rate):
        self.learning_rate = learning_rate #usamos la velocidad de aprendizaje establecida en el constructor de la clase
        self.beta1 = 0 #inicializamos beta1 en 0
        self.beta0 = 0 #inicializamos beta0 en 0
          
    def fit(self, X, y, epochs = 100):
        N = len(X)  #cantidad de filas del dataset
        history = [] #vamos a guardar ac el valor de la funcin de prdida en cada iteracin
        
        for e in range(epochs):  #calculamos el valor de la prdida para cada iteracin
            for i in range(N):                
                Xi = X[i, :]
                yi = y.iloc[i] 
                
                hi = self.beta1 * Xi + self.beta0 #valor predicho para la fila i
                f = hi - yi #residuo o error de prediccin para la fila i
                
                self.beta1 -= self.learning_rate * 2 / N * f * Xi #por definicion de beta0 
                self.beta0 -= self.learning_rate * 2 / N * f # por definicion de beta0 
            
            loss = mean_squared_error(y, (self.beta1 * X + self.beta0))
                                      
            if e % 100 == 0:
                print(f"Epoch: {e}, Loss: {loss})")
            
            history.append(loss) #guardamos los valores de la funcin de prdida en cada iteracin
                                      
        return history
                
    def predict(self, X):
        return self.beta1 * X + self.beta0
